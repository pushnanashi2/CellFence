Machine-checkable boundaries

A manifest-driven repository firewall your agents can see before they edit.

Machine-checkable boundaries for monorepos in the age of Cursor and Claude Code. Prompts explain the architecture. CellFence enforces it in local CLI and required CI checks.

01 / ownership

Cells own paths

03 / ratchets

How it works

Deterministic checks, not another prompt.

CellFence evaluates paths, declared public entries, dependency edges, and ratchet baselines from repository state. The decision path is repeatable in local CLI and CI.

1

Declare governed cells

Map source paths to ownership, public entry points, consumers, resources, and artifact lanes in one manifest.

2

Run deterministic checks

Fail private cross-cell imports, unowned source, undeclared lanes, and dependency drift without calling an LLM.

3

Ratchet expansion

Let architecture shrink without ceremony, but force new edges, entries, and surfaces into review.

CLI required check 
$ npx cellfence check --manifest cellfence.manifest.json

accept ownership map
accept public entry dependencies
reject private import across cells
reject undeclared artifact lane

CI integration

Make the fence visible before merge.

Run CellFence in pull requests and mark it required. The manifest becomes a reviewable contract instead of a paragraph in a team document.

GitHub Actions yaml 
name: cellfence
on: [pull_request]

jobs:
  architecture:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npx cellfence check

Comparison

Different from linters, Nx, dependency-cruiser, and good-fences.

CellFence is focused on manifest-driven repository governance: who owns paths, what is public, which edges are allowed, and what expansion must be reviewed.

CellFence Ownership, public entries, dependency edges, artifact lanes, resources, and ratchets. Designed as a required CI check for agent-edited repositories. Deterministic path and manifest checks. No LLM decision step.
good-fences A TypeScript dependency boundary tool organized around conceptual areas. CellFence adds manifest ownership, public-surface contracts, and ratchet baselines. Use it when repository governance needs one reviewable contract across agents and humans.
Nx / dep-cruiser Great graph and dependency tooling for workspaces and import rules. CellFence is narrower: repository governance as one reviewable contract. Use alongside them when their graph data is useful.
