CellFence Architecture Firewall

Manifest-driven repository governance for TypeScript and JavaScript codebases changed by parallel AI coding agents.

Click & hold

The fence before the edit

Turn architecture from prose into deterministic CI pressure.

CellFence describes repository cells, ownership paths, public entries, artifact lanes, resource contracts, and baseline ratchets. Agents and humans can still move fast, but private imports, undeclared coupling, and silent boundary growth become explicit failures. 

$ npx cellfence baseline check \
  --manifest cellfence.manifest.json \
  --baseline cellfence.baseline.json

reject private cross-cell import
reject undeclared artifact lane
accept public entry dependency

Governed surfaces

The checks are small, sharp, and hard to negotiate with.

01 / ownership

Cells own paths

No overlapping scopes. Strict mode can require every governed source file to belong to exactly one cell.

02 / entrypoints

Public APIs stay public

Cross-cell dependencies must be declared and must target the producer's public entry instead of internal files.

03 / ratchets

Architecture cannot drift quietly

Baselines allow shrinkage, but new cells, edges, symbols, lanes, and signature changes fail until reviewed.
