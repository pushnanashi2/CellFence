# OSS TS/JS 200-Repository Onboarding Run - 2026-07-18

This is a larger public-OSS CellFence onboarding run. It is not a precision
claim and it is not evidence that named upstream repositories contain defects.
All subjects used unreviewed `manifest.strategy: infer`, so the run measures
whether CellFence can clone fixed public commits, infer a non-destructive
starter manifest, execute deterministically, and preserve evidence for later
manual labeling.

No target repository dependencies were installed, no target package scripts were
executed, no upstream issues were opened, and target checkouts were discarded
after manifests and logs were preserved.

## Corpus

Corpus manifest: `docs/research/corpora/oss-ts-js-200-2026-07-18.json`

Selection:

- 200 public GitHub repositories;
- 100 primary-language TypeScript repositories and 100 primary-language
  JavaScript repositories;
- non-archived, non-fork repositories;
- repository `diskUsageKb < 50000`;
- TypeScript query used `stars:>=500`;
- JavaScript query used `stars:>=1000`;
- exact 40-hex default-branch commits were recorded before running checks;
- all subjects used `manifest.strategy: infer`.

The corpus was fixed before the run. Because these are unreviewed inferred
manifests, results are suitable for onboarding and tuning analysis only.

## Bug Found

The first 200-subject run found a CellFence CLI crash on `honojs/hono`:
`Cannot read properties of undefined (reading 'kind')`. The trigger was a
zero-argument SQL-like `.query()` call. The SQL literal detector now accepts an
undefined first argument and the regression is covered by
`tests/resource-access-coverage.test.mjs`.

After the fix, the same corpus completed 200/200 subjects with no tool errors
or unparseable outputs.

## Interpretation

What this run shows:

- CellFence can run across 200 fixed public repositories without installing or
  executing target packages.
- Shallow clone and checkout-discard modes keep local disk usage practical for
  larger solo experiments.
- Real public repositories quickly expose implementation bugs that synthetic
  fixtures missed.
- The evidence bundle is sufficient to start deterministic manual labeling.

What this run does not show:

- It does not establish precision, recall, or false-positive rate.
- It does not prove that any named upstream repository is wrong.
- It does not validate inferred manifests as architecture contracts.
- It does not compare CellFence against Nx, dependency-cruiser, ESLint
  boundaries, CODEOWNERS, or package export maps.

## Labeling Backlog

Next work should use the 651-sample evidence bundle, not the full raw finding
set, and classify each sampled finding as `true_positive`, `false_positive`,
`needs_policy`, `invalid_setup`, or `out_of_scope`.

Priority slices:

- `CELLFENCE_PUBLIC_ENTRY_MISSING`: measure inference fallback rate and improve
  onboarding heuristics.
- `CELLFENCE_PRIVATE_IMPORT`: distinguish real boundary reaches from inferred
  cells that should be merged or translated from existing project boundaries.
- `CELLFENCE_UNDECLARED_RESOURCE_ACCESS`: separate true resource contract gaps
  from test fixtures, build scripts, and intentionally dynamic paths.
- `CELLFENCE_OWNERSHIP_OVERLAP`: manually inspect the 19 findings in
  `eyaltoledano/claude-task-master`, `HeyPuter/puter`, and
  `modelcontextprotocol/servers`.

Only after that pass should this corpus be used in any public precision number.
