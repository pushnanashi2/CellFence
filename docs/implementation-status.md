# Implementation Status

| mechanism | status | enforcement location | limitations | verification method |
|---|---|---|---|---|
| Manifest shape validation | enforced | `@cellfence/schema` and `@cellfence/engine` | JSON schema is implemented by TypeScript validators, not a standalone schema file; v1 rejects unknown object fields and duplicate package/contract names that would overwrite policy maps | fixture tests and `cellfence check` |
| Duplicate cell ID detection | enforced | `@cellfence/engine` | Detects IDs inside one manifest file | invalid fixture |
| Owned path overlap detection | partially_enforced | `@cellfence/engine` | Literal prefix overlap is segment-aware; arbitrary glob intersection remains conservative | invalid fixture and API regression tests |
| Strict governed-source ownership | enforced | `@cellfence/engine` with `manifest.governance.requireOwnership` | Enforces that included source files are owned by a cell and imports cannot target governed unowned source; governed symlinks must resolve inside the owning cell; `cellfence init` enables it, while disabled coverage emits a warning for compatibility | strict ownership and symlink fixtures |
| Public entry and artifact ownership | enforced | `@cellfence/engine` | Public entries and produced artifact lanes must be covered by the declaring cell's `ownedPaths`; glob containment is conservative | ownership fixtures |
| Private cross-cell import rejection | enforced | `@cellfence/engine` | Static analysis only; ES imports, export-from edges, TypeScript `import = require(...)`, selected CommonJS require forms (`require`, `module.require`, simple aliases), and selected `createRequire(...)` aliases are analyzed; computed dynamic imports and computed `require()` calls are fail-closed errors under the built-in required-rule profile; NodeNext runtime `.js` specifiers and tsconfig path aliases, including `extends`, are resolved before boundary checks | invalid fixture |
| Undeclared consumer rejection | enforced | `@cellfence/engine` | Applies to repository-local cells | invalid fixture |
| Public entry existence | enforced | `@cellfence/engine` | One public entry per cell in v0.x | invalid fixture |
| Public symbol match | enforced | `@cellfence/engine` | Export forms are limited to common TypeScript declarations/named exports and common Python `__all__` or top-level declarations | invalid fixture and Python fixtures |
| Artifact lane declaration | enforced | `@cellfence/engine` | File path lanes only in v0.x | invalid fixture |
| Static resource contract declaration | partially_enforced | `@cellfence/engine` | Detects selected string-literal file, SQL table, queue/topic, HTTP patterns, Prisma delegates, TypeORM entities/repositories/query builders, Drizzle table operations, string-literal query builders, BullMQ, KafkaJS, NestJS routes, Fastify routes, FastAPI routes, Django URLConf/model manager calls, SQLAlchemy query/table calls, and Celery task/publish calls; general dynamic dataflow is not inferred | valid and invalid resource fixtures |
| Adapter-scoped resource detection | partially_enforced | `@cellfence/engine` | ORM, query builder, HTTP-framework, and broker-client coverage is per adapter; unsupported libraries require a dedicated adapter or runtime evidence | adapter fixtures and documentation |
| Unresolved resource access fail-closed | enforced | `@cellfence/engine` | Unsafe raw SQL, selected dynamic SQL assembly, TypeORM query-builder dynamic tables, and Drizzle dynamic table arguments fail instead of becoming silent blind spots | invalid dynamic SQL and adapter fixtures |
| Runtime resource evidence | enforced | `@cellfence/schema`, `@cellfence/engine`, and `cellfence evidence check` | Requires explicit evidence JSON; CellFence does not observe live infrastructure by itself | evidence fixtures and CLI tests |
| Runtime trace hook | partially_enforced | `@cellfence/trace` | Node.js fs read/write and fetch tracing plus explicit database/HTTP/queue helper records in v0.x; source-code module loading is ignored | trace tests |
| Plugin API v1 | enforced | `@cellfence/plugin-api` and `@cellfence/engine` | Programmatic rules and adapters are supported through `checkRepository({ plugins })`; manifest `plugins` and `extends` are rejected in v0.x instead of being silently ignored | plugin API tests |
| Official rule plugins | enforced | `@cellfence/plugin-agent-budget`, `@cellfence/plugin-blast-radius`, `@cellfence/plugin-legacy-strangler`, `@cellfence/plugin-quants-trend`, `@cellfence/plugin-geo-purity`, and `@cellfence/plugin-dependency-sovereignty` | Programmatic Plugin API packages; CLI manifest auto-loading is intentionally not enabled in v0.x | official plugin tests |
| Official reporter plugin | enforced | `@cellfence/reporter-economy-matrix` | Renders architecture flow summaries; it is visualization, not an enforcement substitute | official plugin tests |
| Official adapter packages | partially_enforced | `@cellfence/adapter-call-pattern` and `@cellfence/adapter-opentelemetry` | Declarative call-pattern adapter handles static call forms; OpenTelemetry converter imports supplied spans but does not collect traces by itself | official plugin tests |
| Rule severity policy | enforced | `@cellfence/schema` and `@cellfence/engine` | Repository, cell, path override, and CLI severity settings are supported; built-in core rules and manifest `requiredRules` cannot be weakened below error | plugin API tests |
| Resource access baseline inventory | enforced | `@cellfence/engine` baseline metrics | Captures selected static and supplied runtime evidence inventory; arbitrary ORM and broker coupling remain outside v0.x inference | baseline resource fixtures |
| Baseline ratchets | enforced | `@cellfence/engine` | New baselines store accepted cell IDs, ownership sets, public entry paths, public symbol sets, isolated declaration-derived public surface hashes, dependency edges, artifact contracts, and resource inventory; public signature mismatch messages do not print the replacement hash; legacy count fields remain for compatibility | baseline fixture tests |
| Baseline seal | enforced | `@cellfence/engine` and `cellfence baseline create/check/update/sign/verify` | Ed25519 signatures are supported for asymmetric baseline authorization; HMAC remains available for isolated verifier setups; hand-edited files fail when a verifier is configured; locked cells require a configured verifier during `baseline check` | CLI tests |
| Locked baseline update protection | enforced | `@cellfence/engine` and `cellfence baseline update` | Enforces locked cells across count metrics, contract sets, isolated declaration-derived public surface hashes, and resource inventory; locked resource contracts are context and resolution metadata in v0.x | CLI tests |
| Agent context projection | enforced | `@cellfence/engine` and `cellfence context` | Projects manifest and baseline state; it does not grant permissions or approve contract expansion | CLI tests |
| Agent auto-allocation projection | enforced | `@cellfence/engine` and `cellfence context --auto-allocate` | Heuristic task-to-cell matching from manifest text; empty selections require a human-selected cell or clearer task | CLI tests |
| Agent instruction installer | enforced | `cellfence install` | Writes checksumed managed blocks for `AGENTS.md` or `CLAUDE.md`; `install --check` fails on missing, stale, hand-edited, or duplicated unmanaged fence instructions | CLI tests |
| MCP agent server | enforced | `cellfence serve --mcp` | Exposes context, changed-check, claim creation, and finding explanation tools over stdio JSON-RPC; it is a tool surface, not an external authorization boundary | CLI tests |
| Agent claim leases | enforced | `@cellfence/engine` and `cellfence claim create/check/list` | Uses a JSON claim store, defaulting to `.cellfence/claims.json`; local `claim create` writes are serialized with a lock file and atomic rename; cross-machine coordination still depends on a reliable shared claim store | CLI tests |
| Coupling graph output | enforced | `@cellfence/engine` and `cellfence graph` | Emits observed and declared coupling as JSON or Mermaid; visualization is not an enforcement substitute | CLI tests |
| Suggested resolutions | enforced | `@cellfence/engine` JSON findings | Nonbinding; callers choose whether to apply code, manifest, baseline, or human-review paths | CLI tests |
| Stable finding fingerprints | enforced | `@cellfence/engine` JSON findings | Fingerprints ignore human message text and are used by changed-finding diffs and audit summaries | engine and CLI tests |
| Changed findings diff | enforced | `@cellfence/engine` and `cellfence check --changed` | Uses a temporary Git worktree and compares finding fingerprints; base checks receive the same plugins and severity configuration as head checks; it still performs full repository analysis in v0.x | CLI tests |
| History replay research harness | partially_enforced | `scripts/history-replay-study.mjs` | Replays exact before/after commits and introduced finding fingerprints without target installs or scripts; raw replay findings require event labels before precision or upstream-defect claims | history replay script tests |
| History replay mechanism smoke | enforced | `scripts/history-replay-smoke.mjs` | Builds a local exact-commit replay fixture and verifies introduced private-import plus baseline replay findings without relying on external repositories | history replay smoke tests |
| Mutation injection research harness | enforced | `scripts/mutation-injection-study.mjs` | Creates reviewed synthetic repositories and injects known violations to measure controlled recall of expected rules; it is not public-OSS precision or recall evidence | mutation injection script tests |
| Precision evidence pipeline smoke | enforced | `scripts/precision-pipeline-smoke.mjs` | Builds a local reviewed-manifest corpus report, freezes a bundle, injects independent labels, validates hashes, and expects an insufficient-evidence claim decision for the tiny sample | precision pipeline smoke tests |
| Expiring waivers | enforced | `@cellfence/engine` and `cellfence waivers list` | Line-local source comments only; missing expiry, approver, concrete rule, or reason fails the check | CLI tests |
| Waiver request generation | enforced | `@cellfence/engine` and `cellfence waivers request` | Generates approval text only; it never edits source or grants an exception by itself | CLI tests |
| TypeScript/JavaScript source analysis | enforced | `@cellfence/engine` | Static imports, export-from edges, TypeScript `import = require(...)`, selected CommonJS require forms, and exported namespaces are recognized; parser diagnostics, computed imports, and computed CommonJS `require()` calls are reported as unsupported fail-closed findings under the built-in required-rule profile; public surface hashes are isolated declaration-output fingerprints, not full TypeScript semantic-versioning proofs | fixture tests |
| Python source analysis | partially_enforced | `@cellfence/engine` | `.py` files are governed source; imports, public surface, and selected Django/FastAPI/SQLAlchemy/Celery resource patterns are extracted with Python `ast`; unsupported syntax and template-like Python files are fail-closed findings; absolute imports resolve through manifest-derived roots plus common `pyproject.toml`, `setup.cfg`, and static `setup.py` package roots; arbitrary Python framework plugin behavior and dataflow remain outside v0.x static inference | Python import, packaging-inference, public-surface, framework-resource, unsupported-syntax fixtures, and OSS Python onboarding pilot |
| CLI exit codes | enforced | `cellfence` package | Internal errors are grouped under exit code 3 | CLI tests |
| Human-readable, Markdown, JSON, and SARIF output | enforced | `cellfence` package | Markdown is optimized for PR comments; SARIF maps CellFence findings to code-scanning results without changing enforcement semantics | CLI tests |
| GitHub Actions support | partially_enforced | `.github/workflows` and action wrapper | Required-check configuration must be set externally; workflow actions are pinned to commit SHAs with tag comments; action wrapper invokes the published CLI with `npx`; release verification fails if the wrapper's CLI version drifts from the package version; CI includes Linux Node 20 plus cross-platform Node 22 smoke coverage for Linux, macOS, and Windows | workflow files, release verification, and self-check |
| Semantic workflow-control detector v3.2 | partially_enforced | `@cellfence/engine` reference detector and research fixtures | Dependency-free reference implementation freezes workflow deletion and suppression replacement behavior; full YAML-aware protected-path enforcement is a follow-up | `tests/workflow-controls.test.mjs` and research freeze docs |
| npm package smoke verification | enforced | `scripts/pack-smoke.mjs` and CI `pack-smoke` job | Uses local tarballs to validate npm install, CLI execution, README/LICENSE inclusion, and forbidden generated metadata exclusion before registry publish | `npm run pack:smoke` |
